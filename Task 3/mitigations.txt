1. Critical Vulnerabilities (192.168.110.135)

* SSL Version 2/3 Protocol Detection (CVE-2014-3566)
  Risk: Allows POODLE attacks decrypting secure traffic.
  Fix: Disable SSLv2/v3 on all servers.
       Use TLS 1.2+ only (Apache: SSLProtocol -all +TLSv1.2).

*Ubuntu 10.04 End-of-Life (No patches)
 Risk: Unsupported OS with unpatched exploits.
 Fix:Upgrade to Ubuntu 18.04/20.04 LTS immediately.

2. High-Risk Issues

* SMB Signing Not Required (All Hosts)
  Risk: Enables "man-in-the-middle" attacks.
  Fix:Windows: Enable RequireSMB2 and SMB2Signing via Group Policy.
      Linux (Samba): Set server signing = mandatory in smb.conf.

* SWEET32 (Weak SSL Ciphers)
  Risk: Allows brute-force decryption.
  Fix:Disable 3DES/CBC ciphers in Apache/Nginx:SSLCipherSuite HIGH:!aNULL:!MD5:!3DES

3. Medium-Risk Fixes

* Self-Signed/Untrusted SSL Certs
  Fix: Replace with free Let's Encrypt certs or purchase trusted
       certificates.

* SMBv1 Enabled
Risk: Vulnerable to WannaCry/EternalBlue.
Fix:Windows: Disable via Disable-WindowsOptionalFeature -Online -FeatureName smb1protocol.
Linux: Ensure Samba uses min protocol = SMB2.
